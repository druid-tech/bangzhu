GPS 数据

GPS 数据中的各项字段的解释如下表所示。

字段 解释
SN 序列号（Serial Number）
UUID 通用唯一识别码（Universally Unique Identifier），用于计算机体系中识别信息数目的一个 128 位标识符。
Transmitting time 设备与服务器通信的时间，精确到毫秒。
Collecting time 设备采集数据的时间，精确到秒。
Longitude GPS 定位点的经度，精确到小数点后 7 位。
Latitude GPS 定位点的纬度，精确到小数点后 7 位。
Altitude GPS 定位点的海拔高度，单位为 “米”，精确到小数点后 2 位。
Geoid height GPS 定位点的地平高度，单位为 “米”。
Speed 设备移动的速度，单位为 “米”。
Course 从正北方顺时针到设备前进方向之间的夹角度数，数值范围 0 ~ 359.9 （0 表示设备移动方向为正北方）
Satellite 设备在获取 GPS 数据时连接的卫星数。
Positioning mode 定位模式，有效值为 0, 1 和 2。0 表示定位失败，1 表示定位模式为 2D，2 表示定位模式为 3D。
HorAccuracy GPS 水平定位精度，值越低表示水平方向定位越准。
VerAccuracy GPS 垂直定位精度，值越低表示垂直方向定位越准。
Time for Positioning 定位时长，单位为 “秒”。
Data Source 数据采集的来源，有效值为 1, 2 和 4。1 表示计划采集，即设备根据设定的采集频率进行采集的数据。2 表示变频优化，即在计划采集的基础上，包括了以电压作为触发条件的 BOOST 数据。4 表示飞行识别，即在计划采集的基础上，包括了以速度作为触发条件的 BOOST 数据。
